Level 3 - How do we make our model better?

Ok, so you've made it to the boss level. You now understand that for a supervised modeling pipeline, you need something to predict (the target), something to predict it with (features), and something to use to predict (a model). And, we covered the importance of splitting our data to make sure we can understand real life performance, and start to compare performance between models. But, if we simply stop there, we'd be staring at a few different models, maybe can pick the best one, but have no way of making any of them better! And without a chance for and guidance towards improvement, our models might get de-motivated. So let's add a bit to our pipeline:

Buckle up, because we're going to go fast. First, we can consider our target in what we call, target practice. Just kidding, it's probably better called target processing. There are a number of things we could do with our target to get ourselves a better model. We could get more data, especially data that is highly relevant to the prediction task at hand, or data that is novel, or data that is generally hard to predict. If we're trying to predict the final exam scores of Berkeley students in a data science class, we could get data from other final exams from comparable universities in their data science classes. We could also make sure that our data is accurate. Perhaps, in an effort to soften the blow, previous years' final exam scores were curved at different rates. Normalizing those curves, or using pre-curve scores would be a better way to get an accurate model. Again, these considerations uncover and rely on the data generating process behind our target, and can be broadly called target processing.

Second, and you guessed it—if we should pay attention to the target, we should pay attention to the features—we can do some feature engineering. What this means is that we can create new features altogether from the data that we have. Or, we can do some transformations on the features we already have. Or, we can drop some unnecessary or highly similar features (i.e. feature selection) which actually in many cases improves our model performance. Or, we may have to normalize or features or remove outliers. Often, the biggest improvements here come from including orthogonal, or net new, sources of data in your features, especially those that are predictive, correlationally or causally, of the target to some degree.

Third, we can modify the model itself. We can do this by choosing more (or less) complex models. We can modify the free paramaters of each model type (called hyperparameters), which will affect how they train. We can combine a suite of models together (ensemble), or we can stack them on top of each other in layers (stacked). We can use some models to produce intermediate features, like using a computer vision model to produce embeddings of a picture, and use those intermediate features together with other tabular features in a downstream model. The possibilities are endless.

And space (and time to write this) would not permit me to write about cross-validation, alternate metrics, seed hacking, and more. But I have listed the strategies that, in practice, tend to have the largest impact on model performance.
